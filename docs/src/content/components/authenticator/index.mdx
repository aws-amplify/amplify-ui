The `Authenticator` is a simple way to add [authentication](https://docs.amplify.aws/lib/auth/getting-started/q/platform/js)
flows to your application.

## Installation

Install `@aws-amplify/ui-react` with [npm](https://www.npmjs.com/):

```shell
npm install @aws-amplify/ui-react --save 
```

## Getting started

The simplest way to add authentication to your app is to wrap it using `withAuthenticator`:

```js{1,9}
import { withAuthenticator } from "@aws-amplify/ui-react"

function App() {
  return (
    ...
  )
}

export default withAuthenticator(App)
```

## Authenticator with Default Theme

Amplify UI provides simple, clean styles to get started with a great experience in two steps:

1. `import "@aws-amplify/ui-react/styles.css";`
1. Wrap your app with `<AmplifyProvider theme={theme}>` to enable styles.

<Example>

```jsx
import { Authenticator, AmplifyProvider, theme } from "@aws-amplify/ui-react";
import "@aws-amplify/ui-react/styles.css";

<AmplifyProvider theme={theme}>
  <Authenticator>
    {({ state, send }) => (
      <div>
        <Heading>Welcome {state.context.user.username}!</Heading>
        <Button
          className="px-2 bg-white rounded shadow"
          disabled={state.matches("signOut")}
          onClick={() => send("SIGN_OUT")}
        >
          Sign Out
        </Button>
      </div>
    )}
  </Authenticator>
</AmplifyProvider>;
```

## Customizing Styles

All Amplify UI components add `data-amplify-*` attributes that you can target with your CSS selectors.

In this example, `Authenticator` also has a `custom` class added so that other `Authenticator`s on the page aren't affected.

<Example>

```jsx
import { Authenticator } from "@aws-amplify/ui-react";
import "@aws-amplify/ui-react/styles.css";

<style>
  [data-amplify-authenticator].custom [data-amplify-label] { color: red }
</style>

<AmplifyProvider theme={theme}>
  <Authenticator className="custom" theme={theme} />
</AmplifyProvider>
```

## Customizing Components

You can specify your own `components` to customize how Amplify UI renders components.

In this example, three customizations are made:

1. All `Input` primitives are converted to an "underline" style
1. Only the `SignIn`'s submit button is replaced with a "3D" button
1. We replace the

```jsx
<AmplifyProvider
  components={{
    Input(props) {
      return (
        <input
          className="mt-0 block w-full px-0.5 border-0 border-b-2 border-gray-200 focus:ring-0 focus:border-black"
          {...props}
        />
      );
    },
    "Authenticator.SignIn.Button": (props) => {
      return props.type === "submit" ? (
        <button
          className="px-4 py-2 font-bold text-white bg-purple-500 border-b-4 border-purple-700 rounded-md shadow hover:bg-purple-600"
          {...props}
        >
          Custom Sign In
        </button>
      ) : (
        <button {...props} />
      );
    },
    "Authenticator.SignIn.PasswordControl": ({ label }) => {
      return (
        <>
          <Authenticator.SignIn.PasswordControl label={label} />
          <div className="flex items-center">
            <input className="mr-2" type="checkbox" /> Always remember this
            computer
          </div>
        </>
      );
    },
  }}
  theme={theme}
>
  <Authenticator />
</AmplifyProvider>
```
