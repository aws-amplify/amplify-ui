Now that you have Amplify Auth setup, follow the steps below to create an inline policy to enable authenticated app users to access Rekognition.

1. Go to AWS IAM console â†’ Roles

2. Select the newly created `unauthRole` for your project (`amplify-<project_name>-<env_name>-<id>-unauthRole`). Note that `unauthRole` should be used if you are not logging in, but if you are using an authenticator with your application, you will need to use `authRole`.

3. Choose **Add Permissions**, then select **Create Inline Policy**, then choose **JSON** and paste the following:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "rekognition:StartFaceLivenessSession",
            "Resource": "*"
        }
    ]
}
```

4. Choose **Review Policy**

5. Name the policy

6. Choose **Create Policy**

## Best Practices

* End users should complete the Face Liveness check in environments that aren't too dark or too bright and have fairly uniform lighting.
* You should choose a value for the confidence score threshold that reflects the nature of your use case. For use cases with greater security concerns, use a higher threshold.
* You should regularly run human review checks on audit images to check that spoof attacks are mitigated at the confidence threshold you set. 
* You should offer an alternative Face Liveness verification path to your end users if they are photo-sensitive or do not wish to verify their Face Liveness using Rekognition.
