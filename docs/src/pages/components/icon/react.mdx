Icons should be thought of as plain text; they inherit the size and color of their context or can be set directly. Icons themselves do not have any state or handlers, those should be at a higher level like a button. Amplify uses the [Google Material icon set](https://github.com/google/material-design-icons) as the default because it meets the needs above and uses an approved license (Apache-2).

### viewBox

Use the `viewBox` prop to change the SVG [`viewBox`](https://developer.mozilla.org/en-US/docs/Web/SVG/Attribute/viewBox). Both `width` and `height` default to `24`.

```jsx
import { Icon } from '@aws-amplify/ui-react';
import '@aws-amplify/ui-react/styles.css';

// This is a camera icon
<Icon
  ariaLabel="Camera"
  viewBox={{ width: 30, height: 30 }}
  pathData="M10 8v8l5-4-5-4zm9-5H5c-1.1 0-2 .9-2 2v14c0 1.1.9 2 2 2h14c1.1 0 2-.9 2-2V5c0-1.1-.9-2-2-2zm0 16H5V5h14v14z"
/>;
```

### Accessibility

Set an `aria-label` attribute for an Icon.

```jsx
import { IconClose } from '@aws-amplify/ui-react';
import '@aws-amplify/ui-react/styles.css';

<IconClose ariaLabel="Close" />;
```

<Example>
  <IconClose ariaLabel="Close" />
</Example>

### Third-party icon library

Use third-party icon libraries like react-icons.

```jsx
import { FaBeer } from 'react-icons/fa';
import { Button } from '@aws-amplify/ui';

export default function MyButton() {
  return (
    <Button>
      <FaBeer />
    </Button>
  );
}
```
