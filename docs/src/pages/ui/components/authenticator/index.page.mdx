to embed in your application with minimal boilerplate.

## Quick Start

> Preqrequisite: [Create an Authentication backend](https://docs.amplify.aws/lib/auth/getting-started/q/platform/js)

The quickest way to get started is by wrapping your application with the Authenticator component.
This will introduce complete flows for user sign-in and registration and render your app
when users authenticate.

<Fragment>{({ platform }) => import(`./quick-start.${platform}.mdx`)}</Fragment>

## Features

### Sign Up with Username

By default, `Authenticator` creates new users in the Amazon Cognito UserPool
based on a unique `username` using [`Auth.signUp`](https://docs.amplify.aws/lib/auth/emailpassword/q/platform/js).

<Feature name="sign-up-with-username" />

### Confirm Sign Up

If your backend has multi-factor authentication enabled, new sign-ups will [receive a confirmation code](https://docs.amplify.aws/lib/auth/emailpassword/q/platform/js#confirm-sign-up).

<Feature name="confirm-sign-up" />

### Sign In

Users who have registered with their `username`, `email`, or `phone_number` are signed in via [`Auth.signIn`](https://docs.amplify.aws/lib/auth/emailpassword/q/platform/js#sign-in).

<Feature name="sign-in-with-username" />

### Sign In with SMS MFA

If your backend has [Multi-Factor Authentication (MFA) enabled with `SMS`](https://docs.amplify.aws/console/auth/authentication#to-configure-how-users-log-in-to-an-app),
sign-ins will need to verify before authentication.

<Feature name="sign-in-sms-mfa" />

### Sign In with TOTP MFA

If your backend has [Multi-Factor Authentication (MFA) enabled with `TOTP` (Time-based One-Time Password)](https://docs.amplify.aws/console/auth/authentication#to-configure-how-users-log-in-to-an-app),
sign-ins will need to verify before authentication.

<Feature name="sign-in-totp-mfa" />

### Sign In with Social Sign-in

If your backend has [Social Sign-in](https://docs.amplify.aws/lib/auth/social/q/platform/js), users
can sign in with Google, Amazon or Facebook.

<Feature name="sign-in-federated" />

### Internationalization (I18n)

The `Authenticator` ships with [translations](https://github.com/aws-amplify/amplify-ui/blob/main/packages/ui/src/i18n/translations.ts) for:

- `en` – English (default)
- `zh` – Chinese
- `fr` – French
- `de` – German
- `it` – Italian
- `ja` – Japenese
- `es` – Spanish

These [translations](https://github.com/aws-amplify/amplify-ui/blob/main/packages/ui/src/i18n/translations.ts) can be customized using the [Amplify JS' `I18n`](https://docs.amplify.aws/lib/utilities/i18n/q/platform/js/) module:

```js
I18n.setLanguage('fr');

I18n.putVocabularies({
  fr: {
    'Sign In': 'Se connecter',
    'Sign Up': "S'inscrire",
  },
  es: {
    'Sign In': 'Registrarse',
    'Sign Up': 'Regístrate',
  },
});
```
