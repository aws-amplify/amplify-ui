## Getting started

> Preqrequisite: [Create an Authentication backend](https://docs.amplify.aws/lib/auth/getting-started/q/platform/js)

The simplest way to get started is by wrapping your application with the `Authenticator`.
This will introduce complete flows for user sign-in and registration and render your app
when users authenticate.

<Fragment>
  {({ platform }) => import(`./getting-started.${platform}.mdx`)}
</Fragment>

## Features

### Sign Up

By default, `Authenticator` creates new users in the Amazon Cognito UserPool
based on a unique `username` using [`Auth.signUp`](https://docs.amplify.aws/lib/auth/emailpassword/q/platform/js).

<Feature>
  {() => import('@/features/ui/components/authenticator/sign-up.feature')}
</Feature>

### Sign Up with Email

The `Authenticator` automatically infers [`email` as a login mechanism](https://docs.amplify.aws/console/auth/authentication#to-configure-how-users-log-in-to-an-app).

You can explicitly configure Amplify to match your backend via:

```js{4-10} file=../../../../../../examples/next/pages/ui/components/authenticator/sign-up-with-email/index.tsx#L2-L10

```

<Feature>
  {() =>
    import('@/features/ui/components/authenticator/sign-up-with-email.feature')
  }
</Feature>

### Confirm Sign Up

If your backend has multi-factor authentication enabled, new sign-ups will [receive a confirmation code](https://docs.amplify.aws/lib/auth/emailpassword/q/platform/js#confirm-sign-up).

<Feature>
  {() =>
    import('@/features/ui/components/authenticator/confirm-sign-up.feature')
  }
</Feature>

### Sign In

Users who have registered with their `username`, `email`, or `phone_number` are signed in via [`Auth.signIn`](https://docs.amplify.aws/lib/auth/emailpassword/q/platform/js#sign-in).

<Feature>
  {() =>
    import(
      '@/features/ui/components/authenticator/sign-in-with-username.feature'
    )
  }
</Feature>

### Sign In with SMS MFA

If your backend has [Multi-Factor Authentication (MFA) enabled with `SMS`](https://docs.amplify.aws/console/auth/authentication#to-configure-how-users-log-in-to-an-app),
sign-ins will need to verify before authentication.

<Feature>
  {() =>
    import('@/features/ui/components/authenticator/sign-in-sms-mfa.feature')
  }
</Feature>

### Sign In with TOTP MFA

If your backend has [Multi-Factor Authentication (MFA) enabled with `TOTP` (Time-based One-Time Password)](https://docs.amplify.aws/console/auth/authentication#to-configure-how-users-log-in-to-an-app),
sign-ins will need to verify before authentication.

  {() =>
    import('@/features/ui/components/authenticator/sign-in-totp-mfa.feature')
